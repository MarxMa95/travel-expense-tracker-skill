# 数据表结构

## 项目管理表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目名称 | 文本 | 是 | 主键，如“日本游” |
| 旅行目的地 | 文本 | 是 | 目的地描述 |
| 开始日期 | 日期 | 是 | 项目开始日期 |
| 结束日期 | 日期 | 是 | 项目结束日期 |
| 状态 | 单选 | 是 | 未开始 / 进行中 / 已结束 |
| 数据表ID | 文本 | 是 | 关联的消费明细表 ID |
| 数据表AppToken | 文本 | 是 | 消费明细表所在 bitable app token |
| 绑定群聊ID | 文本 | 否 | 当前项目主要服务的群聊 ID |

## 消费明细表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 商户 | 文本 | 是 | 消费商户名称 |
| 金额 | 数字 | 是 | 消费金额，必须为正数 |
| 币种 | 文本 | 是 | `CNY` / `USD` 等 |
| 消费时间 | 日期时间 | 是 | 毫秒时间戳 |
| 消费日期 | 单选 | 是 | 格式 `yyyy/MM/dd` |
| 类别 | 单选 | 是 | 餐饮 / 交通 / 住宿 / 门票 / 购物 / 娱乐 / 其他 |
| 付款人 | 人员 | 是 | 飞书用户 |
| 消费类型 | 单选 | 是 | 个人消费 / 公共 AA |
| 原始消息ID | 文本 | 否 | 用于去重和回溯 |
| 备注 | 文本 | 否 | 存补充说明或更正信息 |

## 类别颜色配置

| 类别 | 颜色值 |
|------|--------|
| 餐饮 | 0 |
| 交通 | 1 |
| 住宿 | 2 |
| 门票 | 3 |
| 购物 | 4 |
| 娱乐 | 5 |
| 其他 | 6 |

## 项目状态颜色

| 状态 | 颜色值 |
|------|--------|
| 未开始 | 7 |
| 进行中 | 4 |
| 已结束 | 0 |

## 去重建议

如果平台支持，优先用 `原始消息ID + 金额 + 商户 + 消费日期` 做幂等保护，避免同一截图被重复记账。
